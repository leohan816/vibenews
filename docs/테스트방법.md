# 테스트 방법

## 타입/린트
```bash
npm run typecheck   # 앱 + 서버(tsconfig) 타입 체크
npm run lint        # 공식 Expo flat config(eslint-config-expo/flat)로 frozen 구현 TS/TSX 범위 lint
```

## 서버 자동화 테스트 (synthetic · 라이브 호출 없음)
모든 provider/HTTP는 주입된 fake transport로만 실행한다. 실제 YouTube/DeepSeek/Fish 호출은 없다.
```bash
npm run test:unit            # 도메인/계약/가드/게이트/캡션/디바이스 머신 단위 테스트
npm run test:integration     # 마이그레이션·provider·재생·API·ops·파이프라인 통합 테스트
npm run test:runtime-local   # migrate CLI 런타임 테스트
```
- 파이프라인 통합 테스트는 fake가 준 **합성 오디오 바이트**를 스테이징에 쓰고, finalization이 그 바이트를
  atomically 이동한 뒤 byte 수와 SHA-256을 재검증하는 실제 저장 경로까지 확인한다(플레이스홀더 바이트 아님).

## 수동 확인 (앱 스켈레톤)
- `npm run web` 실행 후 하단 탭(Listen/Add/Recap/Saved/Settings) 이동 확인.
- Add 탭: 수동 배치(≤10)·채널(≤5) 폼과 파이프라인 상태 표시 확인.
- "재생 중" 화면의 재생/이어듣기가 전역 플레이어 하나로만 동작하는지 확인.

## 라이브 검증
디바이스 A/B/C/D, 실제 오디오, provider 가용성 검증은 독립 구현 리뷰 이후에 수행한다.
