# 변경 기록

사람이 읽는 변경 로그. 최신이 위로.

## 2026-07-11
- **YouTube Add + 전역 이어듣기 MVP (`VN-YOUTUBE-ADD-GLOBAL-RESUME-MVP-001`)** — 프로즌 설계(`설계문서/18`) 구현.
  - **서버(신규)**: private Fastify API(loopback + bearer), better-sqlite3 스키마/마이그레이션, 수동 배치(≤10)·승인 채널(≤5) 취득, 공개 자막 → DeepSeek **Builder → 별도 DeepSeek Verifier**(score≥9.0·no critical·≤2회) → Fish TTS 파이프라인, D-009-A scope/payload 가드와 role-specific HMAC runtime binding, 일일 TTS 상한/영수증/복구, 워커·폴러·백업 오케스트레이션.
  - **오디오 저장/카운트**: 유효한 Fish 오디오 응답을 받은 순간이 successful generation이라 reservation을 release하고 성공 카운트를 올리며 영수증을 `generated`로 남긴다. asset-ready는 **별도**로, 그 실제 바이트를 fsync 스테이징 → atomically 이동 → byte/SHA-256 재검증까지 durable 저장이 검증된 뒤에만 표시한다(저장 실패는 `storage_failed`+defer, 성공 카운트 유지, 자동 재호출 없음, 플레이스홀더 바이트 금지).
  - **앱**: root-lifetime **전역 플레이어 하나**(GlobalPlaybackProvider)로 4상태(unheard/in_progress/completed/skipped) 자동재생·이어듣기, 디바이스 SQLite 저널/아웃박스, Add 탭/컴포넌트, Briefing push-route 전환, Settings SecureStore 토큰.
  - **인큐/무결성**: 수동 아이템·채널 discovery를 atomic하게 claimable job으로 전이(dedupe/scope/멱등), 파이프라인의 실시간 scope 재확인·요청별 attempt 라이프사이클·실제 Verifier 영속화·리스 해제·daily-cap 다음자정 eligibility·TTS 예외안전/재시작 멱등·provenance 메타 갱신 하드닝(Advisor 교차점검 반영).
  - **정리**: 로컬 플레이어 소유 훅(`use-audio-player-controller`) 삭제, 샘플/fallback 오디오 제거, "재생 중" 화면을 전역 플레이어 표면으로 재작성. lint는 공식 Expo flat config로 frozen 구현 범위만 대상.
  - 라이브 YouTube/DeepSeek/Fish 호출과 디바이스 A/B/C/D는 독립 리뷰 전까지 **NOT_RUN**. synthetic 테스트(unit/integration/runtime)만 실행. 상세: [`구현로그/2026-07-11_youtube_add_global_resume_mvp.md`](구현로그/2026-07-11_youtube_add_global_resume_mvp.md).
