# VibeNews docs (구현 기록)

## 이 폴더의 역할
- `docs/`는 **구현 후 기록용**이다.
- 실제 구현 내용, 실행법, 테스트법, 환경변수, 변경기록을 기록한다.
- **설계와 실제 구현이 달라진 경우 반드시 `docs/구현로그/`에 남긴다.**

> 구현 **전에** 정하는 문서는 [`설계문서/`](../설계문서/README.md)에 있다. 역할을 섞지 않는다.

## 구성
- [실행방법.md](실행방법.md) — 앱 실행/개발 서버
- [테스트방법.md](테스트방법.md) — 검증/테스트 방법
- [환경변수.md](환경변수.md) — 사용하는 환경변수(값은 `.env`, 커밋 금지)
- [변경기록.md](변경기록.md) — 사람이 읽는 변경 로그
- [구현로그/](구현로그/) — 블록별 구현 기록 + 설계 대비 차이
