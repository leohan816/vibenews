# 실행 방법

## 앱 (Expo)
```bash
npm install
npm run web       # 웹 브라우저 (가장 빠른 확인)
npm run ios       # iOS 시뮬레이터 (macOS 필요)
npm run android   # Android 에뮬레이터
npm start         # Expo Dev 서버 (QR로 실기기 Expo Go)
```
- 앱은 loopback 뒤 tailnet 전용 API에만 붙는다. `EXPO_PUBLIC_VIBENEWS_API_BASE_URL`에 Tailscale Serve
  HTTPS 주소를 넣고, 디바이스 bearer 토큰은 Settings 화면에서 SecureStore에 저장한다.

## 서버 (private · loopback)
`.env.server.local`(mode-600, 커밋 금지)에 [`환경변수.md`](환경변수.md)의 값을 채운 뒤:
```bash
npm run server:config-check          # 설정 검증(값은 출력하지 않음)
npm run server:migrate               # SQLite 마이그레이션 적용
npm run server:migrate -- --dry-run  # 적용 없이 계획만 확인
npm run server:api                   # private API (127.0.0.1 전용)
npm run server:worker                # 처리 워커(Builder→Verifier→Fish)
npm run server:poller                # 승인 채널 피드 폴러
npm run server:backup                # 온라인 백업 + 무결성 검증
```
- 배포/systemd 유닛은 `ops/systemd/`에 **작성만** 되어 있고 이 구현 pass에서 설치·기동하지 않는다.
- 배포 후 스모크: `node scripts/server-smoke.mjs` (loopback `/v1/health/live`만 확인, provider 호출 없음).

## 라이브 acceptance
실제 YouTube/DeepSeek/Fish 호출과 디바이스 검증은 독립 구현 리뷰 전까지 실행하지 않는다.
`npm run accept:private`는 현재 `NOT_RUN_PENDING_IMPLEMENTATION_REVIEW`만 출력한다.
