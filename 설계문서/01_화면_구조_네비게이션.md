# 01. 화면 구조 · 네비게이션

## 목적
전체 라우팅 구조와 각 화면의 진입 경로·역할을 정의한다.

## 사용자 경험
하단 탭으로 5개 주요 영역을 오가고, 상세/보조 화면은 각 탭에서 push로 진입한다.

## 화면 구성 (Expo Router)
```
src/app/
├─ _layout.tsx                 루트 Stack (탭 그룹 + push 화면들)
├─ (tabs)/
│  ├─ _layout.tsx              하단 탭 네비게이터
│  ├─ index.tsx        Listen     (기본 화면, 큰 중앙 재생 버튼)
│  ├─ briefing.tsx     Briefing   (카테고리/예약 브리핑)
│  ├─ recap.tsx        Recap      (오늘 들은 내용 요약)
│  ├─ saved.tsx        Saved      (뉴스 지식 저장 카드)
│  └─ settings.tsx     Settings   (개인화 설정)
├─ briefing-session.tsx        BriefingSessionScreen  (재생 화면·chapter)
├─ explore-more.tsx            ExploreMoreScreen
├─ daily-recap-detail.tsx      DailyRecapDetailScreen
├─ saved-card-detail.tsx       SavedCardDetailScreen
├─ schedule-briefing.tsx       ScheduleBriefingScreen
├─ foundation-candidate.tsx    FoundationCandidateScreen
├─ product-recommendation.tsx  ProductRecommendationScreen
└─ voice-command.tsx           VoiceCommandScreen
```

## 진입 경로 / 역할
| 화면 | 진입 경로 | 역할 |
| --- | --- | --- |
| Listen | 앱 시작 기본 탭 | 큰 재생 버튼, 오늘 준비된 브리핑 요약 |
| BriefingSession | Listen 재생 버튼 → push | 실제 재생 화면(ambient visual, chapter 이동, 저장/더 알아보기) |
| Briefing | 탭 | 카테고리 브리핑 카드, 예약 브리핑 진입 |
| ScheduleBriefing | Briefing → push | 예약 시간/길이 설정 |
| Recap | 탭 | 오늘 들은 내용 요약 목록 |
| DailyRecapDetail | Recap 항목 → push | 특정 날짜 Recap 상세(음성+텍스트+카드) |
| Saved | 탭 | 카테고리별 뉴스 지식 저장 카드 |
| SavedCardDetail | Saved 카드 → push | 저장 카드 상세 |
| ExploreMore | 재생/카드에서 "더 알아보기" → push | 내부 심층 텍스트 분석 |
| FoundationCandidate | Saved/Settings → push | Foundation 전송 후보 관리 |
| ProductRecommendation | 재생/카드/Recap → push | 개인화 상품 추천(mock) |
| VoiceCommand | Listen/Settings → push | 음성 명령 화면(준비 중) |
| Settings | 탭 | 개인화 설정 |

## Taxonomy / UI 정책 (오늘의 흐름)

VibeNews 핵심 문장: "뉴스/콘텐츠는 공통으로 준비하고, 브리핑은 사용자마다 다르게 조립한다."
이 화면 문서에서 상단 탭과 하단 카드는 의미가 다르므로 명확히 구분한다.

### 상단 가로 탭 = CategoryFilter
- 화면 상단의 가로 스크롤 탭은 **CategoryFilter**다. taxonomy 3축 중 `topicCategory`(큰 주제)를 필터링하는 UI.
- 예시 탭: `전체 | 뉴스 | AI | 건강 | 투자 | 피부관리 | 뷰티 | 비즈니스`
- Category는 "큰 방 8~10개"이며, 아래의 TopicCluster / Tag / Entity와 다르다.

| 개념 | 정의 | UI 위치 |
| --- | --- | --- |
| Category (CategoryFilter) | 큰 방 8~10개 (topicCategory 축) | 상단 가로 탭 |
| TopicCluster (오늘의 흐름) | 브리핑 묶음 = 오늘의 흐름 | 하단 카드 |
| Tag | 세부 주제 | 카드 내 해시태그 |
| Entity | 회사/제품/사람/도구/repo 이름 | 카드/상세 |

### 하단 카드 영역 = 오늘의 흐름 (카테고리 아님)
- 하단 카드 영역은 카테고리가 **아니다**. → **오늘의 흐름 / 태그 묶음 / 브리핑 묶음**을 보여준다.
- 각 카드는 TopicCluster(브리핑 묶음)를 하나씩 표현한다.

### UI 문구 규칙
- UI 노출 문구: "카테고리" → **"오늘의 흐름"**으로 통일한다.
- 내부 데이터/필터 축 이름은 `topicCategory`를 유지하되, 사용자에게 보이는 라벨만 "오늘의 흐름" 계열로 표기.

### 컴포넌트 명명
- `CategoryCard` → **FlowCard / TopicClusterCard** (동의어, `FlowCard`를 표시명으로 사용).
- 상단 탭 컴포넌트는 `CategoryFilter`.
- 구현 시점: **Phase 2**에서 실제 컴포넌트 도입(현재는 mock/future 표기).

### 카드 예시 (FlowCard / TopicClusterCard)
```
AI 에이전트 흐름 · 오늘 7개 · 약 10분 · #Agent #Browser #Claude
```
- 제목: TopicCluster 이름("오늘의 흐름")
- 메타: 오늘 묶인 콘텐츠 개수 · 예상 재생 시간
- 하단: 세부 Tag(해시태그)

관련 정본: taxonomy 타입 정의는 [10_DataModel](10_DataModel_데이터구조.md).

## 오디오 단말기 UI 원칙 (Neo-Retro AI Radio)

홈과 플레이어는 뉴스 피드가 아니라 **오디오 단말기 / 라디오** 느낌이어야 한다.
"AI가 세상의 신호를 잡아 나만의 브리핑 라디오로 들려주는 앱"이라는 정체성을 화면 구조로 드러낸다.
읽는 화면이 아니라 **듣는 기기**를 만진다는 감각(audio-first, less feed / more player)을 우선한다.

### 홈 (Listen) 레이아웃
위→아래 순서로 오디오 단말기의 전면 패널처럼 구성한다.

- **상단 바**: 좌측 로고, 좌측 메뉴, 우측 검색. (텍스트 헤더가 아니라 기기 상단 조작부 느낌)
- **Category 탭**: 상단 가로 스크롤 탭(= CategoryFilter, `topicCategory` 축). 채널 선택 다이얼 감성.
- **상태 pill**: 준비 상태 / mock / 오늘의 믹스 상태를 작은 pill로 표시. (위험은 작은 coral dot만)
- **중앙 Hero Playback Card**: 화면의 주인공. 오디오 단말기의 **메인 패널** 느낌으로, 큰 물리 버튼 같은 play 버튼을 중앙에 둔다. 오늘의 믹스(Today Mix)를 한 번에 재생하는 진입점.
- **하단 오늘의 흐름**: 방송/채널 카드(FlowCard / TopicClusterCard) 목록. 라디오 채널 리스트처럼 배치.
- **Bottom Tab**: 5개 주요 영역(Listen / Briefing / Recap / Saved / Settings).

### 플레이어 (BriefingSession) 원칙
- 음악 재생기 / AI 라디오처럼 보이고 만져진다.
- **텍스트는 최소화**하고, **큰 재생 버튼**을 중심에 둔다.
- chapter 이동·저장·더 알아보기는 보조 컨트롤로, 재생 조작이 화면의 중심.

### 톤
- warm premium retro: 옛 라디오/워크맨의 촉감만 빌리고 UI 자체는 현대적으로 유지.
- 네온/사이버펑크/과한 밝은 색 금지, warm 다크모드 + amber orange 액센트.
