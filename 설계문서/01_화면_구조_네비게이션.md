# 01. 화면 구조 · 네비게이션

## 목적
전체 라우팅 구조와 각 화면의 진입 경로·역할을 정의한다.

## 사용자 경험
하단 탭으로 5개 주요 영역을 오가고, 상세/보조 화면은 각 탭에서 push로 진입한다.

## 화면 구성 (Expo Router)
```
src/app/
├─ _layout.tsx                 루트 Stack (탭 그룹 + push 화면들)
├─ (tabs)/
│  ├─ _layout.tsx              하단 탭 네비게이터
│  ├─ index.tsx        Listen     (기본 화면, 큰 중앙 재생 버튼)
│  ├─ briefing.tsx     Briefing   (카테고리/예약 브리핑)
│  ├─ recap.tsx        Recap      (오늘 들은 내용 요약)
│  ├─ saved.tsx        Saved      (뉴스 지식 저장 카드)
│  └─ settings.tsx     Settings   (개인화 설정)
├─ briefing-session.tsx        BriefingSessionScreen  (재생 화면·chapter)
├─ explore-more.tsx            ExploreMoreScreen
├─ daily-recap-detail.tsx      DailyRecapDetailScreen
├─ saved-card-detail.tsx       SavedCardDetailScreen
├─ schedule-briefing.tsx       ScheduleBriefingScreen
├─ foundation-candidate.tsx    FoundationCandidateScreen
├─ product-recommendation.tsx  ProductRecommendationScreen
└─ voice-command.tsx           VoiceCommandScreen
```

## 진입 경로 / 역할
| 화면 | 진입 경로 | 역할 |
| --- | --- | --- |
| Listen | 앱 시작 기본 탭 | 큰 재생 버튼, 오늘 준비된 브리핑 요약 |
| BriefingSession | Listen 재생 버튼 → push | 실제 재생 화면(ambient visual, chapter 이동, 저장/더 알아보기) |
| Briefing | 탭 | 카테고리 브리핑 카드, 예약 브리핑 진입 |
| ScheduleBriefing | Briefing → push | 예약 시간/길이 설정 |
| Recap | 탭 | 오늘 들은 내용 요약 목록 |
| DailyRecapDetail | Recap 항목 → push | 특정 날짜 Recap 상세(음성+텍스트+카드) |
| Saved | 탭 | 카테고리별 뉴스 지식 저장 카드 |
| SavedCardDetail | Saved 카드 → push | 저장 카드 상세 |
| ExploreMore | 재생/카드에서 "더 알아보기" → push | 내부 심층 텍스트 분석 |
| FoundationCandidate | Saved/Settings → push | Foundation 전송 후보 관리 |
| ProductRecommendation | 재생/카드/Recap → push | 개인화 상품 추천(mock) |
| VoiceCommand | Listen/Settings → push | 음성 명령 화면(준비 중) |
| Settings | 탭 | 개인화 설정 |

## 데이터 모델
라우팅 자체는 데이터 모델 없음. 각 화면이 [10_DataModel](10_DataModel_데이터구조.md) 사용.

## 현재 상태
**mock** — 모든 라우트는 존재하고 이동 가능. 데이터는 mock.

## 구현할 컴포넌트
- `src/app/_layout.tsx` (Stack), `src/app/(tabs)/_layout.tsx` (Tabs)
- 위 13개 스크린 파일
- 공용: `Screen`(스크롤 컨테이너), `StatusBadge`(mock/준비 중 표시)

## 구현 전 확인사항
- 탭 5개 구성 확정: Listen/Briefing/Recap/Saved/Settings

## 나중에 연결될 기능
딥링크, 인증 후 라우팅 가드.
