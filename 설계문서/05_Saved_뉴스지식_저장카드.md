# 05. Saved · 뉴스 지식 저장 카드

## 목적
사용자가 저장한 **뉴스 지식**을 카테고리 중심으로 모아 보여준다. 일반 지식과 구분되는 "뉴스 지식"임을 명시한다.

## 사용자 경험
- 저장 카드는 기본적으로 **카테고리 중심**으로 보여준다.
  - 카테고리: AI · 건강 · 뷰티 · 투자 · 피부관리 · 비즈니스 · VibeNews
- 카테고리 안에서 **시간 필터**: 오늘 · 이번 주 · 전체
- 카테고리 안에서 **지식 유형 필터**: 전체 · Fact · Trend · Insight · Action · Question
- 프로젝트 연결순은 **사용자 기본 UI로 쓰지 않는다.** `relatedProjects`는 내부 메타데이터로만.
- 저장되는 것은 **"뉴스 지식"으로 표시**한다.
- **원문 전체 저장 금지.** 가치 있는 지식 문장 / 요약 / 인사이트 / 출처 메타데이터만 저장.

## 화면 구성
- Saved(탭): `카테고리 탭/칩` → `시간 필터` + `지식 유형 필터` → `저장 카드 목록`
- 시간 필터: 오늘 = 저장일이 오늘 / 이번 주 = 최근 7일 / 전체 = 전부 (`savedAt` 기준으로 실제 필터링).
- 카드: 지식 유형 배지(Fact/Trend/…) + 한 줄 지식 + "뉴스 지식" 라벨 + **Foundation 후보 상태 요약**(내부 저장/후보/동기화됨 등).
- SavedCardDetail: 지식 문장, 요약, 출처 메타, Foundation 후보 상태, "더 알아보기" 진입.

## 데이터 모델
`SavedCard`, `KnowledgeCandidate`, `SourceMeta`, `Category` (→ [10_DataModel](10_DataModel_데이터구조.md))
- 지식 유형: `Fact | Trend | Insight | Action | Question`

## 현재 상태
**mock** — 저장 카드 하드코딩. 필터는 로컬에서 동작.

## 구현할 컴포넌트
- `CategoryFilterBar`, `TimeFilter`, `KnowledgeTypeFilter`
- `SavedKnowledgeCard` ("뉴스 지식" 라벨 + 유형 배지)
- SavedCardDetail 섹션들

## 구현 전 확인사항
- 카테고리 집합 확정.
- 지식 유형 5종 라벨 표기(한/영).

## 나중에 연결될 기능
Foundation 동기화, 실제 지식 추출, 원문 메타만 저장하는 파이프라인.
