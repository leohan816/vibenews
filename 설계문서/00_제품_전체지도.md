# 00. 제품 전체 지도

## 목적

VibeNews의 제품 정의와 전체 구조를 한눈에 잡는 앵커 문서. 나머지 문서는 여기서 갈라진다.

## 제품 정의

VibeNews는 **일반 뉴스 앱이 아니다.** **개인화 오디오 인텔리전스 서비스**다.

- **수집** — 사용자가 관심 있는 최신 뉴스/정보/영상/논문/GitHub/커뮤니티 반응을 모은다.
- **청취** — **짧은 오디오 브리핑**으로 들려준다.
- **정리** — 들은 내용을 **Daily Recap**과 **저장 카드**로 남긴다.
- **축적** — 장기적으로 **Foundation에 "뉴스 지식"**으로 쌓는다.

## 사용자 경험 (핵심)

- 앱을 열면 큰 중앙 재생 버튼과 "Leo님을 위한 뉴스가 준비되어 있어요." 같은 개인화 문구가 보인다.
- 오늘 준비된 뉴스 개수 · 예상 청취 시간 · 카테고리별 개수가 보인다.
- 브리핑은 하나의 긴 오디오가 아니라 **chapter 기반 playlist** 구조다.
- 들은 내용은 Recap과 저장 카드로 남고, 가치 있는 것만 Foundation 후보가 된다.

## 전체 구조

```
하단 탭: Listen · Briefing · Recap · Saved · Settings
재생: Listen → BriefingSession(재생 화면, chapter 이동)
파생: ExploreMore · DailyRecapDetail · SavedCardDetail
      · ScheduleBriefing · FoundationCandidate
      · ProductRecommendation · VoiceCommand
```

## 개인화 브리핑 조립 (핵심 구조)

VibeNews는 뉴스를 사용자마다 새로 만들지 않는다. **공통 1회 생성 + 개인별 조립** 두 층으로 나눈다.

```
Global News Pool (하루 공통, 예: 100개)
   수집 · 본문추출 · 요약 · 스크립트 · TTS 를 공통으로 1회 처리
        │
        ▼  사용자별로 가볍게 조립(metadata/tag/embedding/profile)
Personal Briefing Plan (A: 8개 / B: 7개 …)
   같은 풀에서 선택 → 정렬 → 관련 뉴스 연결 → 공통 오디오 재사용
   (깊은 연결 멘트만 선택적으로 LLM)
```

- **비용 원칙:** 뉴스 생성은 공통, 개인화는 가볍게. 모든 사용자를 위해 매번 새로 요약/TTS 하지
  않는다.
- 자세한 흐름·예시는 [03_Briefing](03_Briefing_예약_카테고리_브리핑.md), 타입은
  [10_DataModel](10_DataModel_데이터구조.md).

## 외부 소스 수집 (Source Adapter)

뉴스 수집은 특정 도구에 종속되지 않는다. core는 **`SourceAdapter` 인터페이스**에만 의존하고, 실제
수집 도구는 **교체 가능한 어댑터**로 둔다.

- **Agent Reach**는 core 의존성이 아니라 **Source Adapter 후보**(1차 정적 점검 기준 사용 후보). 실제
  읽기는 yt-dlp·gh·Jina 등 upstream tool 사용.
- 시스템 변경형 자동설치(`install --env=auto`)·브라우저 쿠키 접근은 금지, 격리
  환경(venv/container)에서만 평가.
- **Source Ingestion Toolkit**: yt-dlp·curl·OCR·ffmpeg·Chrome headless·insane-search 등을 **도구
  어댑터 후보**로 정리(강력한 도구 + 엄격한 정책). Agent Reach·insane-search에 **의존하지 않고**
  민감 기능은 기본 비활성. 상세: `docs/구현로그/2026-07-03_source_ingestion_toolkit_reference.md`.
- 정책: [06_ExploreMore](06_ExploreMore_더알아보기.md) · 인터페이스:
  [10_DataModel](10_DataModel_데이터구조.md) · 평가:
  `docs/구현로그/2026-07-03_agent_reach_evaluation.md`.

## Content Intelligence 개요

VibeNews는 단순 뉴스 앱이 아니라, 여러 소스(YouTube/RSS/웹/GitHub/Reddit/X/이미지/GIF/HTML)를
**Content Intelligence JSON**으로 구조화한 뒤 **Global News Pool → Personal Briefing Plan**으로
조립하는 **개인화 오디오 인텔리전스 서비스**다.

**"뉴스/콘텐츠는 공통으로 준비하고, 브리핑은 사용자마다 다르게 조립한다."**

### taxonomy 3축

모든 콘텐츠는 서로 독립적인 3축으로 분류된다(한 축이 다른 축을 결정하지 않는다).

| 축            | 뜻              | 값 예시                                                                                                                                         |
| ------------- | --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| sourceType    | 어디서 왔나     | youtube · rss · web · github · reddit · x · image · gif · html · manual                                                                         |
| contentKind   | 성격이 무엇인가 | news · analysis · tutorial · opinion · research · community_signal · github_update · product_update · product_detail · document · internal_note |
| topicCategory | 큰 주제         | News · AI · Health · Finance · Skin Care · Beauty · Business · Developer · Science · Lifestyle · Internal                                        |

- **뉴스도 하나의 필터/카테고리일 뿐**이다. 소스가 무엇이든 성격이 뉴스면 `contentKind=news`로
  잡는다.
- 예: **YouTube라도 뉴스성이면** `sourceType=youtube` + `contentKind=news`. sourceType과
  contentKind는 별개다.

### 화면 표현

- 상단 가로 탭 = **CategoryFilter**(큰 방 8~10개, UI 문구는 "카테고리" → **"오늘의 흐름"**).
- 하단 카드 = **TopicClusterCard(=FlowCard)**, 즉 브리핑 묶음인 **"오늘의 흐름"**(TopicCluster).
  Category ≠ TopicCluster ≠ Tag ≠ Entity로 각각 다른 개념이다.

## 긴 영상 브리핑 (제품 원칙)

- VibeNews는 뉴스/RSS 같은 짧은 콘텐츠만 다루지 않는다. YouTube 20~60분 영상·인터뷰·강의·기술/건강/투자 영상도 오디오 브리핑으로 **재구성**한다.
- 원칙: 긴 영상은 '짧게 줄이는 것'이 아니라 '들을 가치가 있는 브리핑으로 재구성'해야 한다. 30분 좋은 영상 기본값 = standard(5~8분).
- 기준·파이프라인은 [14_Video_Briefing_Quality_Strategy](14_Video_Briefing_Quality_Strategy.md).

## Source Pool 개요

VibeNews는 **아무 콘텐츠나 가져오지 않는다.** **Source Pool**(VibeNews가 콘텐츠 후보를 가져오는 전체 원천 목록)에서만 후보를 만든다. MD가 지정한 신뢰 소스와 시스템이 감지한 Hot Topic은 **분리**해서 다룬다(신뢰도·과장 위험이 다르다).

### 소스 4유형

| 유형                    | 뜻                                    | 특징                          |
| ----------------------- | ------------------------------------- | ----------------------------- |
| Editorial Source        | MD/운영자 지정                        | 가장 신뢰                     |
| Hot Topic Source        | 시스템 감지 트렌드                    | 과장 위험, 검증 필요          |
| User Requested Source   | 사용자 URL/주제                       | 개인 관심 반영                |
| Internal Project Source | VibeNews/Cosmile/Foundation/SIASIU 연결 | future                        |

### 흐름

```
Source Pool → SourceCandidate 생성 → scoring
   → (선별) selected 후보만 fetch/analyze
   → Content Intelligence JSON(ContentItem)
   → Global News Pool → Personal Briefing Assembly
```

- **모든 후보를 처리하지 않는다.** score(editorial priority/hot topic/user interest/risk)가 높은 후보만 ContentItem으로 승격해 비용·품질을 관리한다.
- 상세: [15_Source_Pool_and_Editorial_Curation](15_Source_Pool_and_Editorial_Curation.md). 타입 정본은 [10_DataModel](10_DataModel_데이터구조.md).

## 후보 승인 게이트 (Candidate Review)

VibeNews는 **모든 콘텐츠를 자동으로 음성화하지 않는다.** 후보는 사람 승인을 거쳐야 비싼 처리로 넘어간다.

```
SourceCandidate 생성
   → CandidatePreview (가벼운 미리보기)
   → MD/Leo(Admin) 사람 승인
   → 승인분만 full fetch·심층 분석·스크립트 생성
   → DeepSeek 검수 (9/10점 이상만 TTS-ready)
```

- **승인 전 full fetch·TTS 금지.** 미리보기만으로 판단하고, 승인된 후보만 비싼 분석/스크립트/DeepSeek 검수로 넘긴다.
- DeepSeek 자동 수정은 **최대 2회**, 그래도 9/10점 미만이면 human review로 넘긴다.
- 상세: [16_Candidate_Review_and_TTS_Approval_Pipeline](16_Candidate_Review_and_TTS_Approval_Pipeline.md).

## Neo-Retro AI Radio (제품 감성)

VibeNews는 뉴스 앱이 아니라 **"AI가 세상의 신호를 잡아 나만의 브리핑 라디오로 들려주는 앱"**이다. 읽는 피드가 아니라 듣는 라디오가 제품의 중심이다.

- **시각 정체성 = Neo-Retro AI Radio**: warm 다크(warm black/deep charcoal) 배경 + amber orange 주 액센트 + 옛 라디오·워크맨 같은 **오디오 단말기 촉감**을, 현대 AI 오디오 플레이어 UI로 담는다.
- **audio-first**: 읽기보다 듣기가 먼저. 화면은 텍스트를 늘어놓는 곳이 아니라 재생·흐름을 담는 곳이다.
- **less feed / more player**: 무한 피드가 아니라 오늘의 흐름을 재생하는 플레이어가 주역. 촉감만 retro, UI는 현대적으로 유지한다.
- 상세 방향·컬러·용어: [17_Design_Direction_Neo_Retro_AI_Radio](17_Design_Direction_Neo_Retro_AI_Radio.md).
